# URL Postfix Manager 🐱

Chrome-расширение для управления query-параметрами в URL. Сохраняешь часто используемые «постфиксы» (например, `?theme=dark&debug=true`), и одним кликом применяешь их к текущей вкладке.

![Manifest V3](https://img.shields.io/badge/Manifest-V3-blue) ![No deps](https://img.shields.io/badge/dependencies-none-green) ![License MIT](https://img.shields.io/badge/license-MIT-lightgrey)

## Зачем

Часто нужно открывать одну и ту же страницу с разными query-параметрами — фича-флаги, режимы отладки, языковые версии, UTM-метки, A/B-варианты. Вместо того чтобы каждый раз вручную дописывать `?feature_x=true&debug=1` в адресной строке, сохраняешь это один раз и потом применяешь в два клика.

## Возможности

- 💾 **Сохранение постфиксов** с человекочитаемыми названиями
- 🔁 **Умное применение к URL** — существующие параметры с тем же ключом перезаписываются, а не дублируются
- 📊 **Сортировка по частоте использования** — то, чем пользуешься чаще, всегда наверху
- 📁 **Папки** для группировки (например, «Работа», «Дебаг», «UTM»)
- 🖱️ **Drag & drop** постфиксов между папками
- 🔍 **Поиск** по названию и содержимому
- 🌙 **Тёмная тема** с минимумом визуального шума
- 💾 **Локальное хранение** — данные не уходят в облако, живут в `chrome.storage.local`

## Установка

Расширение пока не опубликовано в Chrome Web Store. Установка через режим разработчика:

1. [Скачай](https://github.com/ryazantsevilya/url-postfix-extension/releases) последний релиз расширения.
2. Открой `chrome://extensions/`
3. Включи **Режим разработчика** (тумблер в правом верхнем углу)
4. Нажми **Загрузить распакованное расширение**
5. Выбери папку с расширением

Готово — иконка с котиком появится в тулбаре.

## Использование

### Добавить постфикс
1. Кликни на иконку расширения
2. Нажми `+` в топбаре
3. Введи название (например, `dark mode`) и постфикс (`theme=dark&lang=ru`)
4. Опционально выбери папку
5. Enter или «Сохранить»

### Применить постфикс
Просто кликни по нему в списке — расширение добавит параметры к URL текущей вкладки и перейдёт на новый адрес.

### Папки
- Кнопка **📁** в топбаре — создать новую папку
- **Двойной клик** по названию — переименовать
- **×** справа от названия — удалить (постфиксы внутри переедут в корень, не пропадут)
- **Клик по шапке папки** — свернуть/развернуть
- **Drag & drop** постфикса на папку — переместить туда. На пустую область внизу — переместить в корень

### Форматы постфикса
Поддерживаются все эти варианты — они эквивалентны:
- `theme=dark&lang=ru`
- `?theme=dark&lang=ru`
- `&theme=dark&lang=ru`

### Поиск
Просто начни печатать в поле поиска сверху — фильтрация по названию и по содержимому постфикса в реальном времени.

## Как это работает технически

Расширение использует `URL.searchParams.set()`, поэтому:
-**Если параметра в URL ещё нет** — он добавляется
- **Если параметр уже есть** — значение перезаписывается (без дублирования)

Пример:
```
URL:      https://example.com/page?lang=en&id=42
Постфикс: lang=ru&debug=true
Итог:     https://example.com/page?lang=ru&id=42&debug=true
                                  ^^^^^^^                          (перезаписан)
                                              ^^^^^^^^^^^^         (добавлен)
```

## Разрешения

В `manifest.json` указаны минимальные права:
- `storage` — сохранение списка постфиксов локально
- `activeTab` — чтение URL текущей вкладки
- `tabs` — обновление URL текущей вкладки

Расширение **не** имеет доступа к содержимому страниц, не отправляет данные на внешние серверы, не использует фоновых скриптов.

## Хранение данных

Всё хранится локально в `chrome.storage.local` под ключом `data_v2`:

```js
{
  postfixes: [
    {
      id: "uuid",
      label: "dark mode",
      postfix: "theme=dark",
      count: 42,           // счётчик использований
      createdAt: 1700000000000,
      folderId: "uuid"|null,
      order: 1700000000000
    }
  ],
  folders: [
    {
      id: "uuid",
      name: "Работа",
      collapsed: false,
      order: 1700000000000
    }
  ]
}
```
